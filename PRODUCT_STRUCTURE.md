# BNK 제품 구조 분석

## 공통 키: 차종 + 적용부 + 칼라

모든 마스터와 재고가 **차종(vehicle_code) + 적용부(part_code) + 칼라(color_code)**를 기준으로 일원화되어 있음.

## 규격과 키 관계

**같은 차종+적용부+칼라라도 제품 종류(원자재/반제품/완제품)에 따라 규격이 다름.**

예시: CN7PE / A/REST FRT / NNB
| 구분 | 두폭 | 두께 | 배율 | 폭 | 길이 |
|------|------|------|------|-----|------|
| 원자재(상지) | - | 0.26 | - | 1400 | - |
| 원자재(하지) | - | 0.26 | - | 1400 | - |
| 반제품(표지) | - | 0.26 | 20 | 1400 | - |
| 반제품(프라이머) | - | 2.5 | 20 | 1370 | - |
| 완제품 | 2 | 3 | 20 | 670 | 1070 |

또한 **같은 차종+적용부+칼라+제품종류에서도 규격이 다른 별개 제품이 존재**할 수 있음.

예시: MX5 / UPR RR (4CVT) / NNB — 완제품이 2건
| 두폭 | 두께 | 배율 | 폭 | 길이 | 비고 |
|------|------|------|-----|------|------|
| 1 | 3 | 20 | 1040 | 645 | 규격 A |
| 2 | 3 | 20 | 645 | 1040 | 규격 B (폭↔길이 반전) |

따라서 **제품을 유일하게 식별하려면 규격을 키에 포함해야 함:**
- 일별 재고 키: `차종 + 적용부 + 칼라 + 완제품코드 + 두폭 + 두께 + 배율 + 폭 + 길이 + 공정 + 날짜`

## 마스터 정보

| 구분 | 테이블 | 키 | 추가 구분 |
|------|--------|-----|----------|
| 원자재 | raw_materials | 차종+적용부+칼라 | + 종류(상지/하지) + 두께/폭 |
| 반제품 | master_semi_products | 차종+적용부+칼라 | + 종류(표지/프라이머) |
| 완제품 | master_finished_products | 차종+적용부+칼라 | |

## 공장별 공정 흐름

### 경주 공장 (반제품 생산)

```
상지 (원자재) + 표면처리제/접착제 (원자재) → 표지 (반제품)
폼 (원자재) + 프라이머 (원자재)           → 폼 프라이머 (반제품)

표지, 폼 프라이머 → 납품처 또는 울산 공장으로 이동
```

### 울산 공장 (완제품 생산)

```
표지 (반제품, 경주에서 이동) + 하지 (원자재) + 폼 (미처리 폼 or 폼 프라이머)
  → 엠보 → 재단 (완제품) → 팩킹
```

- 완제품 생산 시 미처리 폼 또는 폼 프라이머 중 선택 사용 가능

## 원자재 공급 업체

| 원자재 | 업체 | 비고 |
|--------|------|------|
| 상지, 하지 | 협성, 한진 | 발주 70% 코오롱, 30% BNK 직접 |
| 폼 (FOAM) | 영보 | BNK 내부 재고 보고 발주 |
| 표면처리제 | 더온 | BNK 내부 재고 보고 발주 |
| 접착제, 프라이머 | 진광 | BNK 내부 재고 보고 발주 |

## 재고 관리 대상

### 경주 공장

| 구분 | 항목 | 재고 창고 |
|------|------|-----------|
| 원자재 | 상지 | 업체 창고 + BNK 경주 창고 |
| 원자재 | 표면처리제/접착제 | BNK 경주 창고 |
| 원자재 | 폼 (미처리) | BNK 경주 창고 |
| 원자재 | 프라이머 | BNK 경주 창고 |
| 반제품 | 표지 | BNK 경주 창고 |
| 반제품 | 폼 프라이머 | BNK 경주 창고 |

### 울산 공장

| 구분 | 항목 | 재고 창고 |
|------|------|-----------|
| 원자재 | 하지 | BNK 울산 창고 |
| 원자재 | 미처리 폼 | BNK 울산 창고 |
| 반제품 | 표지 | BNK 울산 창고 (경주에서 이동) |
| 반제품 | 폼 프라이머 | BNK 울산 창고 (경주에서 이동) |
| 완제품 | 완제품 | BNK 완제품 재고 + 코오롱 완제품 재고 |

## 메뉴 구성

```
재고 관리
├── 재고 현황 (통합 조회 - 읽기 전용, 전 공정 최신 재고 요약)
├── 경주 공장 재고 (탭: 상지 | 표면처리제/접착제 | 폼 | 프라이머 | 표지 | 폼프라이머)
└── 울산 공장 재고 (탭: 하지 | 미처리폼 | 표지 | 폼프라이머 | 완제품)
```

- 경주/울산 각 담당자가 자기 공장 페이지에서 일별 재고 입력
- 재고 현황은 양쪽 데이터를 집계하여 통합 조회

## 재고 관리 DB

| 구분 | 테이블 | 키 | 설명 |
|------|--------|-----|------|
| 원자재 재고 | stock_snapshot_lines | snapshot + raw_material_id | 업체별 원자재 상세 재고 |
| 일별 재고 | daily_inventory | 차종+적용부+칼라+규격+공정+날짜 | 공정 단계별 일별 재고 |
| 재고 현황 | (daily_inventory 집계) | 차종+적용부+칼라+규격 | 전 공정 최신 재고 요약 |

## 생산 계획

- 코오롱에서 3개월치 세부 생산 계획을 월 단위로 제공
- 변경 사항 발생 시 주 단위로 변경된 생산 계획 제공

## 향후 고려사항

- 완제품 마스터를 기준으로 전 공정 제품 리스트 일원화
- 공장 간 이동(경주→울산) 추적
- 업체 창고 재고 ↔ BNK 창고 재고 구분 관리 (상지)
- 코오롱 완제품 재고 ↔ BNK 완제품 재고 구분 관리
