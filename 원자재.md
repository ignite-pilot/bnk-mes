- 원자재 재고 관리
	- 전체 기능 - 재고 목록 보기, 엑셀 다운로드, 원자재 업체 재고 현황 추가, 비엔케이 원자재 재고 현황 추가
	- 원자재 업체 재고 현황 등록
		- 기본 정보 - 원자재 업체, 원자재 창고, 재고 기준일, 수정일자, 수정자
		- 원자재 재고 정보
			- 원자재 정보, 재고 수량
	- 비엔케이 원자재 재고 현황 등록
		- 비엔케이 원자재 창고, 재고 기준일, 원자재, 재고 수량, 수정일자, 수정자
	- 목록
		- 검색 조건
			- 업체 종류 (전체 or 원자재 or  비엔케이), 원자재 업체, 창고 이름, 원자재 (원자재 정보 연결 - 복수 선택 가능), 재고 기준일 (기간)
		- 목록 노출 항목 -> 노출 순서 최신 재고 기준일 순서
			- 노출 방식 : 등록은 원자재 정보 & 재고를 묶음으로 하지만, 노출할 때는 개별적으로 보여준다. 
			- 항목 - 재고 기준일, 원자재, 업체 종류 (원자재 or 비엔케이), 재고 수량 (안전재고 수량), 안전재고 대비 위험도
				- 위험도 : 재고 부족 위험(빨강) , 재고 확보 필요 위험(주황), 안전 (초록),  재고 과잉 (연한 갈색) , 재고 과잉 위험 (다크 브라운)
				- 위험도 판단 기준
					- 원자재 재고 : 원자재 안전 재고를 기준으로 판단
						- 안전재고 기준 15% 내외 - 안전, 15 ~ 50% 미만 - 재고 확보 필요,  50% 미만 - 재고 부족 위험, 15% ~ 50% 이상 - 일부 공급 과잉, 50% 이상
					- 비엔케이 재고 : 원자재 재고와 동일. 추후 1주일 생산 계획 대비 수치와 비교해서 위험도 다시 계산 예정
		- 목록 기능
			- 상세 보기 - 원자재에 링크 연결
			- 원천 데이터 수정 버튼 - 추가했던 원자재 재고 원본을 보여준다. 수정 가능항목 -> 원자재 정보, 재고 수량, 재고 기준일, 원자재 업체
				- 원자재 등록 했을 당시에 정보와 화면으로 보여준다.
			- 삭제 버튼 - 원천 데이터 전체를 삭제 (삭제시 원천 데이터가 묶음으로 삭제된다고 경고)


- 원자재 입고 요청/입고 관리
	- 전체 기능 - 원자재 입고 요청 목록, 원자재 입고 요청
	- 원자재 입고 요청
		- 항목
			- 기본 정보 - 원자재 업체, 입고 희망일
			- 원자재 정보 - N개
				- 원자재 상품
				- 수량
		- 입고 요청 등록시, 해당 원자재 업체 담당자에게 email 발송
			- email 발송은 github에 있는 ig-notification와 연동하도록 하며, https://ig-notification.ig-pilot.com 을 사용
			- SMTP 설정은 aws secret manager에 "prod/ignite-pilot/smtp-naver" 값을 참고해서 발송해줘
	- 목록 
		- 검색 조건
			- 원자재 상품 N개, 입고 상태
		- 목록 - 2가지 Tab으로 구성
			- 입고 요청 Tab
				- 목록
					- 입고 요청일, 원자재 업체, 원자재 종류 개수, 입고 희망일
				- 목록 기능
					- 보기 - 입고 희망일에 링크 연결. 입고 요청한 내용 전체를 보여줌 
					- 입고 요청 취소 - 원자재 입고 상태가 모두 요청 중일 경우만 취소 가능. 한개라도 입고 완료가 된 경우에는 취소 불가. 취소시 담당자에게 요청 취소 email 발송
					- 전체 입고 처리 - 입고 요청한 전체 원자재에 대해서 일괄 입고 처리 -> 담당자에게 이메일 발송
					- 전체 반품 처리 - 입고 요청한 전체 원자재에 대해서 일괄 반품 처리 -> 담당자에게 이메일 발송
			- 입고 상세 현황 Tab
				- 목록 - 검색 조건에 맞는 입고 요청에 각각의 재고 정보를 한개의 Row로 만들어서 보여줌
					- 원자재, 원자재 업체, 입고 요청일, 입고 희망일, 입고 상태 (요청, 취소, 입고완료, 반품)
				- 목록 기능
					- 보기 - 원자재에 링크 연결.
					- 입고 처리 - 해당 원자재에 입고 처리 상태로 변경 -> 담당자에게 이메일 발송
					- 반품 처리 - 해당 원자재에 대해서 반품 처리 상태로 변경 -> 담당자에게 이메일 발송
