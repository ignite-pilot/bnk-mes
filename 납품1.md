# 납품 관리 시스템 스펙

## 개요

납품 관리는 납품사, 제품(완제품/반제품), 창고, 연계 업체를 관리하고 납품 요청~완료까지의 흐름을 처리하는 모듈이다.

---

## 1. 납품사 정보

납품 업무의 중심 엔티티. 완제품/반제품과 M:N 관계로 연결된다.

### 데이터 모델

| 필드 | 필수 | 설명 |
|------|------|------|
| 납품처 이름 | O | |
| 납품 완제품 | - | 완제품 정보와 M:N 연결 (junction table) |
| 납품 반제품 | - | 반제품 정보와 M:N 연결 (junction table) |
| 주소 | - | 다음 주소 검색 연동, 우편번호+주소는 readOnly |
| 연락처 | - | 하이픈 자동 포맷 (저장 시 숫자만) |
| 담당자 | - | |
| 담당자 연락처 | - | 하이픈 자동 포맷 |
| 담당자 email | - | |
| 수정일자 / 수정자 | 자동 | 시스템 자동 기록 |

### 화면

- **공통 기능**: 목록, 등록, 수정, 삭제, 엑셀 다운로드
- **검색**: 납품사 이름 (PC/Mobile 동일)
- **목록 컬럼**
  - PC: 납품처 이름, 주소, 담당자, 담당자 연락처, 기능 버튼
  - Mobile: 납품처 이름, 담당자 연락처, 기능 버튼
- **삭제 제약**: 다른 엔티티(연계 업체, 창고, 납품 요청 등)에서 참조 중이면 삭제 불가

## 4. 완제품 정보

납품 대상이 되는 완제품. 완성차 회사 코드는 외부 API에서 가져온다.

### 데이터 모델

| 필드 | 필수 | 설명 |
|------|------|------|
| 완제품 이름 | O | |
| 완제품 코드 | O | |
| 완성차 회사 코드 | - | ig-config-manager CAR_MAKER API 연동 (SelectDropdown) |
| 차량 코드 | - | |
| 차량 이름 | - | |
| 부위 코드 | - | |
| 부위 이름 | - | |
| 색상 코드 | - | |
| 색상 이름 | - | |
| 두께 / 폭 / 길이 | - | 숫자 |
| 수정일자 / 수정자 | 자동 | |

### 외부 API 연동

- **완성차 회사 코드**: `GET /api/delivery-vehicles` → ig-config-manager의 CAR_MAKER 코드 프록시
- 응답: `{ list: [{ id, name, value, description }] }`
- `value`를 DB에 저장, `name (value)` 형태로 드롭다운 표시

### 화면

- **공통 기능**: 목록, 등록, 수정, 삭제, 엑셀 다운로드
- **검색**: 완제품 이름, 완제품 코드 (PC/Mobile 동일)
- **목록 컬럼**
  - PC: 완제품 이름, 완제품 코드, 차량 이름, 부위 이름, 색상 이름, 기능 버튼
  - Mobile: 완제품 이름, 완제품 코드, 기능 버튼

## 6. 완제품 입고요청 / 납품 관리

납품사에 제품 납품을 요청하고, 품목별로 납품/반품을 처리하는 핵심 화면.

### 데이터 모델 — 납품 요청 (부모)

| 필드 | 필수 | 설명 |
|------|------|------|
| 납품사 | O | 납품사 FK (SelectDropdown) |
| 납품 요청일 | O | date |
| 납품 희망일 | O | date |
| 상태 | 자동 | 하위 품목 상태에 따라 자동 계산 |
| 수정일자 / 수정자 | 자동 | |

### 데이터 모델 — 품목 (자식, N개)

| 필드 | 필수 | 설명 |
|------|------|------|
| 유형 | O | 완제품 / 반제품 (SelectDropdown) |
| 제품 | O | 선택한 납품사에 연결된 제품만 노출 (SelectDropdown) |
| 수량 | O | 1 이상 정수 |
| 납품 상태 | 자동 | 등록 시 비노출, 상세에서 표시 |

### 납품사별 제품 필터링

- 납품사를 선택하면 해당 납품사의 `finished_product_ids`, `semi_product_ids`를 API에서 조회
- 품목의 제품 드롭다운에는 연결된 제품만 표시
- 납품사 변경 시 기존 선택된 제품 초기화

### 상태 체계

**납품 요청 상태** (부모):

| 상태 | 코드 | 조건 |
|------|------|------|
| 납품 요청 | `requested` | 모든 품목이 `requested` |
| 부분 납품/반품 | `partial` | 일부 품목만 납품 또는 반품 |
| 전체 납품 | `completed` | 모든 품목이 `delivered` 또는 `returned` |
| 전체 반품 | `all_returned` | 모든 품목이 `returned` |
| 납품 요청 취소 | `cancelled` | 요청 자체를 취소 |

**품목 납품 상태** (자식):

| 상태 | 코드 | 설명 |
|------|------|------|
| 납품 요청 | `requested` | 요청 중 |
| 납품 | `delivered` | 납품 완료 |
| 반품 | `returned` | 반품 처리됨 |
| 납품 취소 | `cancelled` | 요청 취소 시 자동 처리 |

### 화면

- **공통 기능**: 목록, 등록, 수정, 납품 요청 취소, 엑셀 다운로드
- **검색**: 납품사 이름 (PC/Mobile 동일)
- **목록 컬럼**
  - PC: 납품사, 납품 요청일, 납품 희망일, 상태, 품목 수, 기능 버튼
  - Mobile: 납품사, 납품 희망일, 기능 버튼
- **수정/취소 조건**: 모든 품목이 `requested` 상태이고 요청이 `cancelled`가 아닐 때만 가능
- **상세 보기**: 품목 테이블에서 개별 납품 완료 / 반품 버튼 제공

## 공통 UI 패턴

| 패턴 | 설명 |
|------|------|
| SelectDropdown | 커스텀 드롭다운 (검색 지원, 6개 이상 자동 검색) |
| ItemSelectPopup | M:N 관계 선택 팝업 (검색, 전체 선택/해제) |
| 주소 검색 | Daum Postcode API, 우편번호+주소 readOnly |
| 연락처 포맷 | 입력 시 하이픈 자동 삽입, 저장 시 숫자만 |
| 페이지네이션 | 서버 사이드 (page, limit), PAGE_SIZES: 10/15/20/50/100 |
| 엑셀 다운로드 | 현재 검색 조건 반영, CSV 형식 |
| 반응형 | `useIsMobile()` 훅으로 PC/Mobile 컬럼 분기 |
| Soft Delete | `deleted CHAR(1) DEFAULT 'N'`, 실제 삭제 안함 |
